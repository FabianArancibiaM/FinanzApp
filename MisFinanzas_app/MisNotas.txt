Blanco (#FFFFFF): Para mantener el diseño limpio y fácil de leer.
Naranja Suave (#F39C12): Agrega un toque de energía y atención, útil para resaltar alertas o notificaciones importantes.

Estructura y Distribución de la Pantalla
1. Barra Superior (Header)
Título: “Agregar Transacción” centrado en la parte superior.
Botón de Cierre (X): En la esquina superior derecha, para salir fácilmente.
2. Selector de Tipo de Transacción (Ingreso o Gasto)
Dos botones de selección en la parte superior:
Ingreso: Botón a la izquierda con color Verde Esmeralda (#2ECC71).
Gasto: Botón a la derecha con color Naranja Suave (#F39C12).
Al seleccionar uno, cambia el color y la información se adapta según la opción.
3. Campo de Cantidad
Un campo de entrada prominente para la cantidad de la transacción.
Diseño: Un cuadro blanco o gris claro (#ECF0F1) con bordes suaves.
Texto placeholder: “Ingresa el monto” en un gris tenue.
Si es un gasto, el borde o el fondo puede cambiar a Naranja Suave; si es ingreso, a Verde Esmeralda.
4. Categoría de la Transacción
Un menú desplegable para seleccionar la categoría (como "Alimentación", "Transporte", "Entretenimiento", "Salario", etc.).
Iconos para cada categoría: Iconos visuales junto a cada categoría para hacerlo más intuitivo (por ejemplo, un carrito de compras para alimentación, un coche para transporte).
Color: Utiliza íconos en Azul Marino (#2C3E50) sobre un fondo claro para consistencia visual.
5. Fecha de la Transacción
Campo para seleccionar la fecha, que por defecto será la actual pero se puede modificar.
Botón de calendario o selector de fecha con un ícono pequeño junto al campo.
Color: Texto en Gris Oscuro (#7F8C8D) para mantenerlo discreto.
6. Comentarios o Notas (Opcional)
Un campo de texto para agregar una breve nota o descripción adicional.
Placeholder: “Añade una nota…” con texto en gris tenue.
Diseño: Campo blanco con bordes sutiles en Gris Claro.
7. Método de Pago
Un menú para seleccionar el método de pago (tarjeta, efectivo, cuenta bancaria, etc.).
Iconos pequeños para ayudar a identificar visualmente el método.
Color: Fondo claro y texto en Azul Marino para mantener la consistencia.
8. Botón de Confirmación
Botón grande al final de la pantalla, centrado, con texto claro como “Agregar Transacción”.
Color: Si es un ingreso, el botón será Verde Esmeralda (#2ECC71); si es un gasto, será Naranja Suave (#F39C12).
Texto: Blanco para alto contraste y legibilidad.
9. Botón de Cancelar (Opcional)
Justo debajo del botón de confirmación, un pequeño botón de texto como “Cancelar”, en Gris Oscuro (#7F8C8D), para evitar sobresaturar la interfaz visualmente.
Interacciones UX
Validación en Tiempo Real: A medida que el usuario ingresa la cantidad o selecciona la categoría, el botón de confirmación puede activarse o cambiar de color.
Transiciones Suaves: Al seleccionar "Ingreso" o "Gasto", el cambio de color del botón de confirmación y de otros elementos debe ser suave, para evitar distracciones.
Flujo de Interacción
El usuario abre la pantalla "Agregar Transacción".
Selecciona si es un Ingreso o Gasto.
Introduce el monto y selecciona una categoría.
(Opcional) Agrega una nota y selecciona la fecha.
Elige el método de pago.
Confirma la transacción con el botón Agregar Transacción.
